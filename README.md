1.Briefly summarize the requirements and goals of the app you developed. What user needs was this app designed to address? 
 
WeightTrack is a weight tracking app designed for android devices. The purpose of this app is to allow users to keep track of their weight each day to help achieve their goals. I wanted to make it as easy and simplified as possible to allows all types of users to be able to use and enjoy my app. 
 
2. What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful? 

WeightTrack has 3 main screens: Login screen to sign in, a weight log screen to show all entries, and a screen to allow for SMS permissions. Each UI provides a crucial element for my app to function for my users. The UI was left intentionally simple to minimize confusion and to focus on my user needs. 
 
3. How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future? 
 
The app was mainly built with 4 repositories each with separate classes for its database. This allowed for the code to be easily read and debugged. I used the readings heavily throughout the entire process of creating this project. For future android projects, I will most likely continue to use repositories the way I did, as they are good for persistent local data storage. 
 
4. How did you test to ensure your code was functional? Why is this process important, and what did it reveal? 
 
I mainly tested using the android emulator while developing. After I created each feature, I would test to see if there were any bugs. Doing these tests in increments showed that it consolidated problems, making it quicker and easier to fix. 
 
5. Consider the full app design and development process from initial planning to finalization. Where did you have to innovate to overcome a challenge? 
 
I probably had the most issues with getting the SMS feature to work. I was unable to get SMS set up at login, so instead I had it on its own screen.  
 
6. In what specific component of your mobile app were you particularly successful in demonstrating your knowledge, skills, and experience? 
 
I would say that I was most proud of my databases implantation. It correctly defines users, weights, and goals with SQLiteOpenHelper.
